# lizzys-lab
the canonical home of everything. if it's not in here, it isn't real. if it is, it's live.

## one-time setup (~30 min total, then frozen until october)
1. **GitHub:** create a private repo named `lizzys-lab` at github.com/new, then upload this folder's contents (drag & drop works on github.com — no terminal needed).
2. **Netlify:** Site settings → Build & deploy → link this GitHub repo. Publish directory: `sites`. From now on, every commit auto-deploys. Delete the drag-and-drop habit.
3. **Claude Code (desktop):** open it, point it at this folder. It reads CLAUDE.md automatically — it will already know the brand. First command to try: "read CLAUDE.md and the launch doc, then tell me what's due this week."

## the map
- `CLAUDE.md` — the constitution. every AI session starts here.
- `/sites` — everything Netlify serves: the Shrine (`ritual.html`), the Wall (`wall.html`), the Booth (`booth.html` — note: its AI engine only runs inside Claude, the hosted copy is a reference).
- `/brand` — the 90-day launch doc. the plan is frozen; ideas go to the runway.
- `/content/runway.md` — every new idea lands here under a pillar. nothing gets built from it until a Sunday review says so.
- `/noseblind` — drop NOSEBLIND_Pilot_v5.docx and the series bible here. september = packaging only.
- `/.claude/skills` — recipes Claude Code runs on command.

## the freeze rule
this system is DONE as of july 2026. no new layers, no new tools, no restructuring until the october quarterly review. building infrastructure feels like progress. it is not a post, a product, or a filed LLC.
